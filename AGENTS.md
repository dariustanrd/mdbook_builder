## Cursor Cloud specific instructions

This is an **mdBook** static site project (no npm/pip/cargo project files). The only development dependency is the `mdbook` CLI binary.

### Services

| Service | Command | Port | Notes |
|---------|---------|------|-------|
| Dev server | `mdbook serve --hostname 0.0.0.0 --port 3000` | 3000 | Watches for file changes and live-reloads |

### Key commands

- **Build:** `mdbook build` (outputs to `book/`)
- **Serve (dev):** `mdbook serve --hostname 0.0.0.0 --port 3000`
- See `README.md` for full documentation on project structure and workflows.

### Caveats

- There are no automated tests, no linter, and no CI checks beyond the GitHub Actions build workflow.
- The landing page tries to fetch `catalog.json` at runtime. In local dev this file doesn't exist (it's generated by CI), so the landing page shows "Unable to load book catalog" — this is expected behavior locally.
- The Admin page's "Add New Book" section is hidden until a GitHub token is provided and "Load Catalog" is clicked. To inspect the form UI without a token, remove the `hidden` class from `#catalog-section` via browser DevTools.
- `mdbook` is installed via pre-built binary to `~/.local/bin`. Ensure `~/.local/bin` is on `PATH`.
